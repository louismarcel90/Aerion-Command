# Product Vision

## Project Name

AERION COMMAND

## Long Name

Deterministic Tactical Air Combat, Replay & Mission Assurance Simulator

## One-Line Positioning

AERION COMMAND is a deterministic tactical air combat simulator with replay, explainability, degraded operations, and evidence-grade mission analysis rendered through a premium ASCII terminal interface.

## What This Is

AERION COMMAND is a simulation system.

It models tactical air combat as a mission-oriented environment where aircraft, sensors, weapons, threats, commands, and mission objectives evolve through deterministic simulation ticks.

The project is intentionally built to demonstrate high-discipline software engineering in a compact, interactive, and visually engaging form.

## What This Is Not

AERION COMMAND is not:

- a casual terminal game
- a physics-accurate flight simulator
- a military training product
- a black-box AI combat simulator
- a visual-only demo
- a random arcade dogfight

## Product Promise

The user should feel that every action has weight.

The reviewer should see that every important system decision has structure, traceability, and a clear boundary.

## Experience Goals

The terminal experience must feel:

- fast
- clear
- tactical
- readable
- responsive
- satisfying
- replayable

The engineering experience must feel:

- modular
- deterministic
- testable
- documented
- explainable
- reliable under faults

## Reviewer Promise

A reviewer should be able to understand three levels of the project:

### 30-Second Level

A premium ASCII tactical air combat simulator with mission replay.

### 3-Minute Level

A deterministic simulation system with separated state, rendering, event history, and scenario packs.

### 15-Minute Level

A mission-assurance architecture with invariants, degraded modes, replay verification, reason-coded outcomes, and evidence pack export.

## Differentiation

The rare part of AERION COMMAND is not ASCII rendering alone.

The rare part is the combination of:

- deterministic tick simulation
- mission-oriented state progression
- explainable tactical outcomes
- replay verification
- degraded system handling
- evidence generation
- polished terminal experience
