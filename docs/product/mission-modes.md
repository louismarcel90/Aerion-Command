# Mission Modes

AERION COMMAND supports multiple modes to make the project useful, replayable, demonstrable, and technically credible.

## 1. Training Mode

Training Mode teaches the basic control model.

It introduces:

- speed changes
- heading changes
- climb and descent
- radar focus
- target lock
- weapon authorization
- countermeasures
- mission alerts

## 2. Tactical Mission Mode

Tactical Mission Mode is the main interactive mode.

Mission types include:

- intercept
- escort
- defend
- survive
- threat containment

## 3. Replay Mode

Replay Mode reconstructs a mission from event history and command sequence.

It supports:

- play
- pause
- step forward
- snapshot jump
- event inspection
- replay verification

## 4. Debrief Mode

Debrief Mode explains what happened.

It includes:

- mission timeline
- key events
- score breakdown
- threat windows
- missed opportunities
- outcome reasons
- evidence references

## 5. Degraded Systems Mode

Degraded Systems Mode introduces controlled uncertainty.

Examples:

- radar degradation
- reduced track confidence
- intermittent RWR
- partial HUD
- delayed warnings

## 6. Fault Injection Mode

Fault Injection Mode tests robustness.

Examples:

- dropped command
- delayed event processing
- sensor jitter
- lock loss
- replay checksum mismatch
- stale track propagation

## 7. Evidence Export Mode

Evidence Export Mode produces a mission evidence pack.

The evidence pack includes:

- scenario identity
- seed
- command digest
- event log digest
- final state digest
- mission outcome
- replay verification status
- outcome reason summary

## 8. Recruiter Showcase Mode

Recruiter Showcase Mode is a short, high-impact mission designed for quick demonstration.

It must show:

- terminal HUD
- tactical map
- radar track
- lock attempt
- missile authorization
- event feed
- debrief summary
- replay/evidence capability
